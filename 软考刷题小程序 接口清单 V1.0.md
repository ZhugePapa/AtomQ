### 1.3 接口风格约定

- 协议：`HTTPS`
- 数据格式：`JSON`
- 字符编码：`UTF-8`
- 时间格式：默认使用 `ISO 8601`
- 成功响应结构统一为：`code`、`message`、`data`
- 写接口需返回明确业务错误码

### 1.4 通用响应结构

```json
{
  "code": 0,
  "message": "ok",
  "data": {}
}
```

### 1.5 通用错误码

| 错误码 | 含义 | 说明 |
| :--- | :--- | :--- |
| 0 | 成功 | 请求成功 |
| 40001 | 参数错误 | 缺少参数或参数格式错误 |
| 40003 | 资源不存在 | 章节、题目、知识点等不存在 |
| 40009 | 状态冲突 | 当前状态不允许该操作 |
| 40101 | 未登录 | 需要登录后访问 |
| 40301 | 无权限 | 非会员访问会员内容 |
| 40901 | 重复提交 | 幂等冲突或重复交卷 |
| 42901 | 请求过频 | 超过频控限制 |
| 50001 | 系统异常 | 服务端异常 |

## 2. 鉴权与请求头

### 2.1 鉴权方式

- 游客接口：不要求登录态
- 登录接口：通过微信登录换取业务 `token`
- 登录后接口：请求头带 `Authorization: Bearer <token>`
- 会员能力：在登录态基础上校验会员权益

### 2.2 通用请求头

| Header | 是否必填 | 说明 |
| :--- | :--- | :--- |
| Authorization | 否 | 登录接口后必填 |
| X-Platform | 是 | `wechat-miniapp` |
| X-App-Version | 是 | 小程序版本号 |
| X-Trace-Id | 否 | 请求链路追踪 ID |
| Idempotency-Key | 写接口推荐 | 幂等请求唯一键 |

## 4. 用户与鉴权

- 接口：`POST /api/v1/auth/login`

| 字段 | 类型 | 必填 | 说明 |

## 12. 联调重点

### 12.1 前后端联调顺序

1. 登录与首页
2. 章节目录与知识点详情
3. 今日一练与章节练习
4. 交卷与结果页
5. 错题本
6. 会员与支付

### 12.2 风险点

- 游客示例题额度控制要在服务端兜底
- 交卷幂等必须依赖 `idempotency_key`
- 错题本和掌握度更新要避免重复计算
- 会员内容鉴权要在内容接口和题单接口双重校验

### 12.3 测试重点

- 未登录访问正式练习时返回 `40101`
- 非会员访问会员内容时返回 `40301`
- 重复交卷时返回同一结果或明确的 `40901`
- 断网重试后交卷结果一致
