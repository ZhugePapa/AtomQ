Emotional Design / Minimal Loop

软考刷题小程序 IP 角色设计稿

这份设计稿基于“最小闭环”生成，只聚焦最值得先做的情感化资产： `1 个主 IP + 5 个状态姿态 + 3 张场景化应用方向`。 风格保持理性、可靠、低打扰，服务于学习陪伴、解析反馈和任务推进。

角色关键词

稳定
专业
清晰
陪伴

建议角色名：`项小蓝` 角色定位：学习领航员 / 计划推进官

Main Character

主 IP：项小蓝

1. 头部像一块简洁的信息面板，代表“清晰的信息传达”。

2. 身体采用圆角几何块面，降低拟人感，保留专业和稳定。

3. 主色是高饱和蓝，保证在蓝灰白界面里能承担视觉锚点。

4. 道具不是装饰，而是学习卡、旗帜、进度环等任务语义元素。

Character Spec

角色设定与使用规范

Personality

低打扰、可靠、会给下一步

Tone

不卖萌，不喊口号

Shape

几何化、圆角化、轻科技感

Role

学习领航员 / 项目助理

适用页面

启动页：建立第一印象

学习首页：承担任务指引

解析页：承担修复型反馈

错题本：承担“清空压力”的节奏提示

完成态：承担完成反馈与轻庆祝

Five States

5 个最小闭环状态姿态

平静待命

用于首页、空状态和普通任务区，表达“我在，下一步很清楚”。

任务指引

用于首页任务卡和启动页，强调“今天从这里开始”。

复盘提醒

用于解析页和错题页，表达“这题可以修正，不是失败”。

轻量庆祝

用于完成态和连续学习节点，表达“今天推进得不错”。

耐心陪伴

用于空状态和计划更新完成后，强调“我还在，继续就好”。

Scene 01

学习首页任务陪伴

Today Plan

今天先学 1 节，再做 10 题

用小尺寸 IP 陪伴任务卡，增强“今天从哪里开始”的方向感。

Scene 02

解析页修复反馈

Analysis

这题不是不会，是把概念边界记混了

角色不强化“错误”，而强化“解释清楚和继续修复”。

Scene 03

完成态轻庆祝

今天的主任务已经清空

接下来可以轻量复习，也可以直接休息。

完成反馈只做“轻庆祝”，避免过度游戏化。

Delivery Scope

最小闭环落地清单

1 个主 IP

项小蓝，作为全局情感载体。

5 个状态姿态

平静、指引、复盘、庆祝、陪伴。

3 张场景应用

首页、解析页、完成态优先接入。

1 套文案语气

少评价，多给下一步动作提示。
