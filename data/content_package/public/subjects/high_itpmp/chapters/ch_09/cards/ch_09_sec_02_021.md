### 概述

需求文件中的需求可以按来源、目标和性质分成不同类别。

### 核心内容

| 类别 | 含义 |
|:---|:---|
| ==业务需求== | 整个组织的高层级需要，如解决业务问题或抓住机会 |
| ==干系人需求== | 干系人的需要 |
| ==解决方案需求== | 产品、服务或成果必须具备的特性、功能和特征 |
| ==功能需求== | 产品应该执行的行动、流程或能力 |
| ==非功能需求== | 可靠性、保密性、性能、安全性、服务水平、可支持性等 |
| ==过渡和就绪需求== | 从当前状态过渡到将来状态所需的临时能力，如数据转换和培训 |
| ==项目需求== | 项目需满足的行动、过程或条件，如里程碑日期、合同责任、制约因素 |
| ==质量需求== | 确认可交付成果成功完成的条件或标准 |

### 易混点

性能、安全、可靠性通常是非功能需求；数据转换和培训常属于过渡和就绪需求。
