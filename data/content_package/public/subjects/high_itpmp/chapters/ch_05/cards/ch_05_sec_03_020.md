### 概述

面向对象设计原则用于控制类职责、扩展方式、依赖方向和对象之间的知识范围。

### 核心内容

| 原则 | 核心判断 |
|:---|:---|
| ==单一职责原则== | 一个类只承担单一功能 |
| ==开闭原则== | 对扩展开放，对修改封闭 |
| ==李氏替换原则== | 子类可以替换父类 |
| ==依赖倒置原则== | 依赖抽象，不依赖具体实现 |
| ==接口隔离原则== | 多个专门接口优于单一总接口 |
| ==组合重用原则== | 尽量用组合而不是继承复用 |
| ==迪米特原则== | 对其他对象了解尽可能少 |

### 易混点

迪米特原则与低耦合思想一致；单一职责原则与高内聚思想一致。
