### 概述

UML 图不建议死背长清单，考试更常按“图的用途”考判断。

### 核心内容

| 图 | 主要用途 |
|:---|:---|
| ==用例图== | 描述用例、参与者及其关系 |
| ==类图== | 展现类、接口、协作及关系，常用于静态设计视图 |
| ==对象图== | 描述类图中事物实例的静态快照 |
| ==顺序图== | 强调消息的时间顺序 |
| ==通信图== | 强调收发消息对象的组织结构 |
| ==状态图== | 描述状态、转移、事件和活动 |
| ==活动图== | 描述控制流和数据流，适合业务流程建模 |
| ==部署图== | 描述运行时处理节点及构件部署 |

### 易混点

顺序图强调“时间顺序”，通信图强调“对象组织结构”。
